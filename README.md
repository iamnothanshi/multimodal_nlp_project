# Multimodal Sentiment Analysis on CMU-MOSEI

A course project exploring whether text, audio, and visual modalities convey consistent sentiment signals, benchmarking unimodal and multimodal fusion models on the CMU-MOSEI dataset.

**Report:** [`report/NLP_project_Hanshi.pdf`](report/NLP_project_Hanshi.pdf)

---

## Research Question

> Does the visual modality say the same thing as the text?

We train and evaluate nine models across three categories: text-only baselines (Ridge, SVR, BiLSTM, Transformer), unimodal audio/visual models, and three multimodal fusion architectures (Early Fusion, Late Fusion, Low-Rank Tensor Fusion). We then measure cross-modal disagreement as a function of sentiment intensity.

---

## Results

All experiments use seed = 42. Evaluation is on the held-out test set (~4,659 utterances).

| Model | MAE ↓ | MSE ↓ | Pearson r ↑ | Acc-2 ↑ | F1 ↑ |
|---|---|---|---|---|---|
| Ridge (text) | 0.714 | 0.845 | 0.559 | 0.788 | 0.788 |
| SVR (text) | 0.662 | 0.763 | 0.615 | 0.789 | 0.788 |
| BiLSTM (text) | 0.617 | 0.691 | 0.662 | 0.794 | 0.788 |
| Transformer (text) | 0.648 | 0.742 | 0.634 | 0.783 | 0.775 |
| BiLSTM (audio) | 0.814 | 1.164 | 0.241 | 0.640 | 0.606 |
| BiLSTM (vision) | 0.808 | 1.154 | 0.251 | 0.643 | 0.628 |
| Early Fusion | 0.640 | 0.739 | 0.638 | 0.787 | 0.778 |
| **Late Fusion** | **0.611** | **0.676** | **0.675** | 0.808 | 0.807 |
| **LRTF** | 0.617 | 0.690 | 0.673 | **0.812** | **0.810** |

Key finding: text–vision prediction disagreement grows with sentiment intensity (Pearson r = 0.45), rising from a mean of 0.34 for near-neutral utterances to 1.29 for strongly emotional ones.

## Dataset

This project uses **CMU-MOSEI** ([Zadeh et al., ACL 2018](https://aclanthology.org/P18-1208/)), accessed through the [MultiBench](https://github.com/pliang279/MultiBench) toolkit.

The raw data file (`mosei_senti_data.pkl`) is **not included** in this repository due to size.

To reproduce, download MultiBench and place the data file at:

```
external/MultiBench/datasets/affect/pack/mosei/mosei_senti_data.pkl
```

Pre-extracted features used:
- **Text:** GloVe 840B 300-dimensional embeddings, padded to 40 tokens
- **Audio:** COVAREP (74-dimensional), acoustic and prosodic features
- **Visual:** FACET (35-dimensional), facial action units and head pose

## Reproducing Results

Run the notebooks in order, from 01 to 04.

Notebooks 02 and 03 save model weights to `models/` and metrics to `results/`. Notebook 04 loads from `results/` and requires no GPU.

Pre-computed `results/*.pkl` files are included, so notebook 04 can be run standalone to inspect all analysis and figures.
